HTML BASICS: Tables

Property Description
<table> This element creates the base for a table
<thead> This element is used for syntax: describes a part of the table that is a heading
<tbody> This element is used for syntax: describes a part of the table that is the body of the table
<tfoot> This element is used for syntax: describes a part of the table that is the footer of the table
<th> This element creates creates a heading inside the table heading
<tr> This element creates rows inside the table
<td> This element allows you to add data to your table row
<colspan> This attribute allows the table data to occupate a determined number of columns
<rowspan> This attribute allows the table data to occupate a determined number of rows

CSS BASICS: Position

Property Description
static The element is positioned according to the normal flow of the document. The top, right, bottom, left, and z-index properties have no effect. This is the default value.
relative The element is positioned according to the normal flow of the document, and then offset relative to itself based on the values of top, right, bottom, and left. The offset does not affect the position of any other elements; thus, the space given for the element in the page layout is the same as if position were static.
absolute The element is removed from the normal document flow, and no space is created for the element in the page layout. The element is positioned relative to its closest positioned (meaning not static) ancestor (if any) or to the initial containing block. Its final position is determined by the values of top, right, bottom, and left.
fixed The element is removed from the normal document flow, and no space is created for the element in the page layout. The element is positioned relative to its initial containing block and always stays in the same place even if the page is scrolled. Its final position is determined by the values of top, right, bottom, and left.
sticky The element is positioned based on the user's scroll position. A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport, then it "sticks" in place (like position: fixed).
